# Systemd

Path: `backend/deployments/systemd`

## Purpose

Boot integration for AgentCore local/server hosts. Units are **generated by the CLI**, not checked in as static host-specific files.

## Operator commands

```bash
# System unit (needs root to write /etc/systemd/system)
sudo "$(command -v agentcore)" boot enable
sudo "$(command -v agentcore)" boot disable

# User unit (no root; enable linger for true headless boot)
agentcore boot enable --user
loginctl enable-linger "$USER"
agentcore boot disable --user
```

The unit is oneshot + `RemainAfterExit` and calls:

- `ExecStart=…/agentcore service start` — Compose `postgres`/`neo4j` + MCP HTTP daemon
- `ExecStop=…/agentcore service stop`

Day-to-day process control (without changing boot policy):

```bash
agentcore service start
agentcore service status
agentcore service restart
agentcore service stop
```

## Modular Boundary

This directory is part of the AgentCore backend modular architecture. Host unit files are written at enable-time under `/etc/systemd/system/agentcore.service` or `~/.config/systemd/user/agentcore.service`.
